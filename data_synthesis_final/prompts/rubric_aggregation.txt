# Task Instructions
Please execute the merge following this strict protocol:

### 1. Aggregation & Analysis
- Analyze each criterion against the original `User Prompt` to ensure relevance.

### 2. Conservative Deduplication Strategy (CRITICAL)
Apply a **Conservative Merging Strategy**. Do NOT merge items merely because they look similar.

- **MERGE ONLY IF** the semantic meaning, required action, and scope are 100% identical.
- **DO NOT MERGE** if they differ in granularity, thresholds, parameters, or implied method.

### 3. Conflict Resolution & Refinement
- **Wording**: Prefer professional, concise, unambiguous phrasing.
- **Weights**: If merging items with different weights, keep the higher weight.
- **Binary Standard**: Ensure every `description` is binary (True/False) and discriminative; avoid vague wording.

# Output Requirements (Most Important!)
- Return ONLY a JSON array wrapped in a Markdown ```json code block```.
- No extra text before/after the code block.
- Schema for each item:

  - `description`: string (binary, verifiable)
